# dominion_app

Flutter personal finance app (expenses, income, goals, obligations, persons) using `flutter_riverpod`, `go_router`, `dio`, `drift` (local SQLite via `drift_flutter`), and `freezed`/`json_serializable` codegen.

## Running

- Devices: macOS desktop, Chrome, and Android emulator `Medium_Phone_API_36.1`.
- `flutter emulators --launch Medium_Phone_API_36.1` then `flutter run -d emulator-5554`.
- Android SDK, AVD data, and Gradle cache live on the external drive (`/Volumes/MacBackup/Android/`) via symlinks — see the global CLAUDE.md "Android/Flutter Dev Environment" section. Make sure `/Volumes/MacBackup` is mounted before building.
- `lib/data/local/database.dart` has a `part 'database.g.dart'` for drift codegen; the generated file is already checked in/present, but re-run `dart run build_runner build` if you change table definitions.

## Known gotchas

- `connectivity_plus` is used in `lib/widgets/connection_indicator.dart` but easy to miss adding to `pubspec.yaml` if dependencies get regenerated by hand — it's already declared, just noting why it's there.
- Riverpod `Notifier.build()` must not synchronously mutate `state` before returning (Riverpod throws "Bad state: Tried to read the state of an uninitialized provider"). `AuthNotifier.build()` in `lib/providers/auth_provider.dart` kicks off its async init via `Future.microtask(_checkAuth)` rather than calling `_checkAuth()` directly, since `_checkAuth`'s body runs synchronously up to its first `await`. Follow the same pattern for any other `Notifier` that needs to self-initialize from async work.
